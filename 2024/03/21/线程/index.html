关于线程

作为一个刚学计算机的小白，我对线程的理解，只能说是一塌糊涂，因为线程实在是有
点奇怪。为什么这么说呢，首先我们看一串代码，与他的执行结果，有意思的事情就出
现了。
‘’’ 

            python
            import time
            from threading import Thread


            def say_hello():
                print('Hello')
                time.sleep(1)
                # 在sleep的时候进行的其他操作
                print("你"+"好", end="啦")


            # 主进程
            if __name__ == '__main__':
                for i in range(5):
                    # 创建子线程
                    t = Thread(target=say_hello)

                    t.start()
                    # t.join()

=========================================================


为什么产生这个结果呢

程序在执行的时候并不是跑完一个任务之后再开始执行另一个的，感觉是在执行程序的
时候，他在跑完一行代码的时候，另一行代码也就开始跑了，这里并不是指睡眠一秒，
而是for 循环的第二个变量在执行时的第一行代码，这也就解释了为什么是五个hello。

这里注意后面的代码，在睡眠一秒后，出现了数据乱码的情况，在print 后的end 有时
执行的慢，有时执行的快，这也解释了线程中的抢跑情况，目前我只能这样理解啦。

希望在python上更进一步啦
